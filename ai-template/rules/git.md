# Git Workflow Rules

Use small, focused commits with clear intent.

## Baseline

- One logical change per commit.
- Write meaningful commit messages.
- Avoid mixing refactor and feature behavior changes in one commit.
- Keep pull requests/review units reasonably small.
- Rebase or merge from target branch regularly to reduce integration conflicts.
