# ai-config Configuration Reference

## Scope

This document describes the practical configuration contract for `./.ai`.

## Root Files

Required by `validate`:
- `.ai/.aiignore`
- `.ai/README.md`
- `.ai/config.yaml`
- `.ai/modules.yaml`
- `.ai/qa.yaml`
- `.ai/manifest.yaml`

## `manifest.yaml`

System metadata and ownership markers. Generated by `init`.

Important fields:
- `schema_version`
- `generator: ai-config`
- `managed_by: ai-config`
- `selected_agent`
- `ui_locale`
- `qa_required_on_start`
- QA lifecycle markers:
  - `qa_version`
  - `qa_completed`
  - `qa_completed_at`

## `config.yaml`

Global runtime behavior defaults.

Key sections:
- `schema.version`
- `profile` (selected bootstrap profile)
- `context` (priority and source behavior)
- `conflict_resolution`
- `behavior`:
  - `task_mode`
  - `questionnaire_on_init`
- `modules`:
  - `registry_file`
  - `strict_enabled_only`
- `output` (warnings/provenance/confidence behavior)
- `safety` (destructive action policy)

## `modules.yaml`

Central module registry.

Each module entry:
- `name`
- `path`
- `enabled`
- `state`: `disabled|bootstrap|ready|degraded`
- `managed_by`

Readiness validation applies to enabled modules.

## `qa.yaml`

Questionnaire source and answers for context hydration.

Recommended:
- keep only actionable, non-duplicated questions
- store answers near questions
- let agent ask only missing/incomplete questions

## Cross-Module Consistency Checks

- `skills/registry.yaml` -> `required_modules` must exist in module registry.
- `orchestration/workflows/*` -> `role` and `fallback_role` must exist in `agents/registry.yaml`.
- locale sanity: for `ui_locale=ru`, QA questions should be in Russian.

## Dependency Rules

- `skills` requires `rules` and `templates`
- `orchestration` requires `agents`
- enabled MCP providers require `mcp`

Behavior:
- interactive init auto-fixes dependencies
- non-interactive init rejects invalid dependency combos
