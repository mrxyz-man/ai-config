# ai-config User Guide

## Purpose

`ai-config` bootstraps and maintains a unified AI configuration layer in `./.ai` for any software project.

Goals:
- bootstrap a stable configuration structure once
- keep it synchronized with template evolution
- let AI agents use `./.ai` as the single source of project context and operating rules

## Core Commands

## `init`

Creates `./.ai` from `./ai-template`.

Recommended:

```bash
npx @mrxyz/ai-config@latest init --cwd <project-path>
```

Non-interactive (CI/bootstrap automation):

```bash
npx @mrxyz/ai-config@latest init \
  --cwd <project-path> \
  --non-interactive \
  --agent codex \
  --ui-locale en \
  --profile standard
```

Key options:
- `--force` recreate managed `./.ai`
- `--agent` `codex|claude|both|other`
- `--ui-locale` locale for user-facing AI content
- `--profile` `minimal|standard|full`
- `--modules` CSV override
- `--task-mode` `off|assisted|enforced`
- `--questionnaire-on-init` `true|false`
- `--enable-mcp-providers` CSV list (requires `mcp` module)

## `validate`

Validates `./.ai` integrity, required root files, YAML correctness, enabled-module readiness, and key cross-module links.

```bash
npx @mrxyz/ai-config@latest validate --cwd <project-path>
```

## `sync`

Synchronizes managed `./.ai` with template updates.

Dry run:

```bash
npx @mrxyz/ai-config@latest sync --cwd <project-path>
```

Apply safe actions:

```bash
npx @mrxyz/ai-config@latest sync --cwd <project-path> --no-dry-run
```

Conflicts-only view:

```bash
npx @mrxyz/ai-config@latest sync --cwd <project-path> --conflicts-only
```

## Preflight States

`init/sync/validate` evaluate project state:
- `fresh`: `./.ai` missing
- `managed`: generated by `ai-config`
- `foreign`: `./.ai` exists but not managed by `ai-config`
- `mixed`: managed markers plus foreign/legacy markers

Expected behavior:
- `init` bootstraps (or recreates with `--force`)
- `sync` works only with `managed`
- `validate` expects `managed`

## What `./.ai` Contains

Root files:
- `.aiignore`: context exclusions
- `README.md`: local config guide
- `config.yaml`: global behavior defaults
- `modules.yaml`: module registry (`enabled`, `state`, `managed_by`)
- `qa.yaml`: question/answer context source
- `manifest.yaml`: system metadata and ownership markers

Agent bridges (in project root):
- `AGENTS.md` (Codex/general)
- `CLAUDE.md` (Claude)

## Module Catalog (Current)

- `project`: project goals, structure, domain context
- `rules`: coding/process/text-locale rules
- `agents`: role definitions and responsibilities
- `mcp`: MCP providers, policies, scenarios
- `skills`: reusable AI capabilities
- `orchestration`: task workflows and status boards
- `contracts`: module-level interface contracts
- `checklists`: PR/release checklists
- `adr`: architecture decision records
- `governance`: ownership and escalation policies
- `interfaces`: integration interfaces and shared schemas
- `runbooks`: incident/release operational procedures
- `risk`: risk register and mitigation plan
- `quality`: quality gates and quality policy
- `security`: baseline controls and secrets policy
- `observability`: telemetry baseline and SLO policy
- `templates`: reusable templates for artifacts
- `memory`: long/short-term project memory
- `logs`: policy + runtime logs

`core` and `qa` are always foundational.

## How AI Agent Should Work After Init

Recommended flow:
1. Load `./.ai/manifest.yaml` and `./.ai/config.yaml`.
2. Respect `ui_locale` for all user-facing text and questions.
3. If `qa_completed: false`, generate and ask only missing questions from `qa.yaml`.
4. Fill/update target files according to answers.
5. Mark QA completion in manifest when required fields are complete.
6. Use enabled modules only (`modules.yaml` + `strict_enabled_only`).

## Recommended User Workflow

1. Run `init`.
2. Review generated `./.ai` quickly (especially `config.yaml` and `modules.yaml`).
3. Enable needed modules in `modules.yaml`.
4. Run `validate`.
5. Start AI-assisted work with `./.ai` as source of truth.
6. After upgrading `ai-config`, run `sync` (dry-run first, then apply).

## Troubleshooting

Common issues:
- `foreign` state: existing custom `./.ai` without ai-config markers.
- `mixed` state: overlapping legacy and managed structures.
- parse error in YAML: `sync --conflicts-only` shows recommendations.
- locale mismatch: for `ui_locale=ru`, QA text must be Russian.

## Versioning and Safety

- Template updates are additive-first in `sync`.
- Conflicting files are not force-overwritten by default.
- Destructive actions are blocked by default via config safety settings.
